# 前言

> 本书以“无人机任务运营的数据闭环工具链”为主线组织全部项目：从外场任务最常见的起飞前检查、飞行判定、日志输出、任务简报等真实场景出发，先用最少的 Python 语法做出“能跑、能用、能看见结果”的脚本，让学生快速建立编程信心与工程感。你将反复练习把任务需求翻译成清晰的程序流程：输入是什么、计算/判定什么、输出什么，以及如何用规范的代码风格把过程表达出来。

> 在此基础上，本书进一步把“任务结果”推进到可交付、可复盘的层面：围绕飞行日志与巡检数据，系统训练字符串、列表、字典、集合等数据结构的处理方法，并通过迭代、批处理与文件/目录操作，将数据整理成稳定的输出物。每个项目都会形成可检查的产出（如简报文本、数据摘要、归档文件），让学生理解无人机应用中“数据从产生到保存再到复用”的完整链路，逐步建立“任务侧—数据侧”的闭环意识。

> 当项目数量增加、机型与规则变化增多时，脚本式写法会出现重复、难维护与难扩展的问题。本书在最后引入面向对象方法，作为对前面成果的“工程化升级”：把全局规则与单机状态分别组织为类属性与实例属性，用实例方法/类方法/静态方法统一自检、配置与工具处理，并通过继承、重写与封装承接不同机型差异。最终，所有项目共同汇聚为一个可持续迭代的总产出——任务数据包（Mission Package）：每次任务都能标准化生成检查单、简报、清洗日志与归档文件，并可由“飞控对象模型”稳定地生成报告，从而实现“能用 → 可交付 → 可扩展”的无人机应用开发闭环。
